Using the caDSR Sentinel Tool > Report Details Tab
Report Details Tab
The Report Details tab contains the properties controlling heading content and distribution.
Content
Defines the detail appearing on the report.
Include information …
Select to include an expanded header at the top of the report. The expanded header contains all the information appearing on the Properties tab in addition to the standard information which includes the reporting date and the source database. Maximum number of levels …
Set the stopping point on reporting the “Associated To” records. Each change in the caDSR has a defined area of effect. The record containing the change is referred to as level 0 (zero). Records directly associated to it are a level 1 (one). Records associated to the associated records are a level 2, and so on.
For example, a Value Domain is changed, it is level 0. The Data Elements referencing the Value Domain are a level 1. The Modules, Class Scheme Items, etc. that reference the Data Elements are a level 2. And so it continues throughout the database. Contexts appear at many levels due to the direct association between the Context and other records.
Continuing the previous example, the Value Domain is directly associated to a Context and so it will be a level 1. The Data Elements associated to the Value Domain are directly associated to Contexts and so it is also a level 2. This can also be visualized as a series of rings originating at the changed record and expanding out.
Distribution
Define when and to whom reports are sent.
Send a report from …
Choose whether a report is sent with every processing of the Alert Definition or sent only when there is activity. To reduce e-mail traffic, select “… only when there is activity …”.
Recipients
The e-mail distribution list for Automated and Manually generated reports. There are no limits on the list size or content. Add a recipient e-mail address by selecting a User Name, a Curators group or typing the address. A Process URL may also be entered.
Users
The user accounts defined within the caDSR. The list displays the user name, suffixed first by an asterisk when the account is missing the e-mail address and suffixed by the associated Curator group names as appropriate within parenthesis. If the e-mail address is missing the user may still be added as a Recipient. When the account profile is updated with an e-mail address, the user will begin receiving alert reports.
Curators
The Curator groups derived from the user account profiles. A user is included in a Curator group when the account is given any write permissions to the caDSR. Read only accounts are not included in any group. Selecting a Curator group as a recipient hides changes in the organization or assigned responsibilities. Alert report e-mails to Curator groups are sent to all members except those listed below “Additional e-mail address”.
Additional e-mail address and Process URL
A desired recipients well formed e-mail address or software well formed URL. This is often used to specify alternate e-mail addresses, temporary recipients and software process applications. The user account profile supports a single e-mail address. Additional e-mail addresses must be entered here, e.g. for John Smith to receive an alert report at both work and home, the user account profile may contain one and the other entered here. Alternatively, the user account profile may not have an e-mail address and both are entered here. If entering a process URL, please contact the NCI Help Desk for programming design requirements and instructions on the target server application.
The following users only receive …
The users who have intentionally requested exclusion from the Recipient Curator groups. These users only receive alert report e-mails when included by name or by e-mail address in the Recipient list.
Once a selection is made or e-mail address is entered, select “Add” to insert the address into the recipients. Select a recipient and “Remove” to stop alert report e-mail distribution to an address.
During the Automated Generation process, alert reports are bundled into a single e-mail per address.
To:
A sample e-mail “To:” address displaying the selected recipients.
Subject:
The “Subject:” appearing on alert report distribution e-mails.

Text entered in this field appears in the report header. It does not appear in the distribution e-mails. As noted above, a recipient may have multiple report references in a single e-mail, consequently the text is included in the report. HTML tags are permitted within the text.
