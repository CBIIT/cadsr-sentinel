Viewing/Sorting Alert Definitions
The Alerts page information is displayed in columns. Use the Show All/Show Private button to list all configured caDSR alerts or only your alerts (respectively).
By default the alerts appear in ascending order by Name, however you can change the sorted order of the alerts by clicking on the header of any column except Summary.
Clicking once on the header of a column sorts the alerts in ascending order based on that column. Clicking twice on the header sorts the list in descending order. Once the list is sorted by a particular characteristic, a small pointer icon appears to the right of the column header used for the sort. If the icon points downward, the sort is ascending; if the icon points upward, the sort is descending.
You can also create a more complex sort order by selecting multiple column headers in reverse sequence. For example, to sort the list by Name and then by Last Auto Run, first select the column heading Last Auto Run and then select the column heading for Name. Use a single or double-click on each column header to sort in ascending or descending order, respectively.
Sorting the columns can be useful if you have selected Show All and need to find alerts based on their name or the user who created them.
The columns on the Alerts List page provide the following information about each alert:
Name
this is the name given to the alert by the alert creator on the Properties Tab. Alert names are not unique (alerts are given a unique, hidden identifier) but should effectively describe the characteristics or purpose for the alert.
Summary
The information shown in the Summary column is generated from the information provided by the creator in theCriteria Tab and the Monitors Tab of the Alert Definition. This column is for overview purposes only and cannot be used for sorting.
Frequency
The Frequency is indicated by the alert creator on the Properties Tab and indicates how often the alert is to be processed by the Automated Alert Report Generator.
Last Auto Run
The Last Auto Run date is set by the Automated Alert Report Generator and indicates the last time the Alert Definition was processed. This date is set whether or not a report is generated. (The Report Details Tab allows you to select whether or not to create a report if there is no activity.)
If this column displays (not yet run), it means that the Alert Definition was created but has not yet been processed by the Automated Alert Report Generator.
The Last Auto Run date/time information applies only to the automated processing of alerts and is not affected by running an alert manually.
Active
The Active column indicates whether the Alert Definition is active or inactive. Active alerts are indicated by a checkmark; inactive alerts are indicated by an x. The active/inactive status of alerts is set on the Properties Tab.
You can use the information in the Frequency, Last Auto Run and Active columns to determine the “currency” of an alert. For example, an alert appears with a Frequency of “Daily” and the Active column contains a checkmark, but the Last Auto Run date/time indicates that the last time the alert was processed was more than a month ago.
You can click the Create Using Button to view the details of the alert, and notice that the Start and End dates are set and the Active period ended on the date of the Last Auto Run.
Creator
The Creator column shows the Login ID of the Alert Definition creator.
