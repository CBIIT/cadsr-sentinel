Using the caDSR Sentinel Tool > Monitors Tab
Monitors Tab
The Monitors tab contains selections that control the triggers for the alert report.
The selection combination on the Monitors tab triggers the alert report creation by interrogating the data activities. When the selections are set to Any Change, all activity within the scope defined on the Criteria tab will trigger creation of the report. This is the default setting for all new Alert Definitions.
Selecting specific values for one or more of the statuses significantly limits the data activities triggering report creation because a specific activity matching the selected value(s) must be found in the caDSR history for the reporting dates. For example, selecting the Workflow Status “RELEASED” will only trigger a report when this specific change is found in the history - other changes would not trigger a report creation.
The lists on this page support multiple selections using the standard Ctrl+Select or Shift+Select method.
Workflow Status(es)
Trigger an alert report when the Workflow Status is changed to one of the selections, or any change if that option is selected. Selecting Ignore causes the monitor to ignore any changes to this field as a trigger for the report.
Registration Status(es)
Trigger an alert report when the Registration Status is changed to one of the selections, or any change if that option is selected. Selecting Ignore causes the monitor to ignore any changes to this field as a trigger for the report.
Version
Trigger an alert report when the Version is changed in a way that meets the selected version trigger:
•
Any Change: Triggers a report for any and all value changes to the record property.
•
Ignore: Disables triggers for alert report creation for the record property.
•
Whole Number Change: Applies to Version number changes only, triggers a report when the record whole number part changes, decimal number part changes are ignored.
•
Specific: Applies to Version number changes only, used to indicate a specific version number to trigger the report creation.
Some data is not logged in the caDSR history and will appear as “Details not available” on an alert report.
