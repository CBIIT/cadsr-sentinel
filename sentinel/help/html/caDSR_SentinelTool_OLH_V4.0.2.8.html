Using the caDSR Sentinel Tool > Properties Tab
Properties Tab
The Properties tab contains the basic Alert Definition properties.
Alert Name
The alert name identifies the Alert Definition and is used both in the notification e-mail and the body of the generated report. Names should be brief but should accurately describe the function or purpose of the alert so that you (or other users) can easily identify an alert on the Alerts Page. For example, Alert Definition names like “Changes to CCR” and “Watch for Retired Records” provide quick-glance, useful information regarding the alert.
Summary
Provides an overview of the alert configuration information. The summary window shows the selections made on the Criteria and Monitors tabs, allowing you to quickly review this information without selecting those tabs. Any reports generated based on the current report definition will only include information that meets the description shown in the Summary window.
Auto Run Options
Allows you to select how often the Alert Definition is run against the caDSR (Frequency) and to select whether the definition is active or not (Status).
•
The Frequency selected more specifically identifies the execution cycle in the Automated Report Generation Process. This process runs daily at a time determined by the System Administrator.
Alerts marked with a frequency of Daily are evaluated with each run. Alerts marked as Weekly are evaluated only on the day of the week identified. Alerts marked Monthly are evaluated only on the specific day of the month identified. To indicate the first day of the month, select 1. To indicate the last day of the month, select 31.
An alert may be run manually at anytime using the Run button. Performing a manual run has no affect on the automated report generation process.
•
The Status selected controls the Auto Run Frequency application of the Automated Report Generation Process.
•
Active, no end date - The Alert Definition is processed according to the Frequency until the alert status is set to Inactive.
•
Active until the first report … After sending a report containing some activity, the status automatically changes to Inactive.
•
Active only from … The Alert Definition is processed only within the dates specified. If you leave the begin date blank, the Alert Definition creation date is used. If you leave the end date blank, the alert will be processed beginning on the begin date and continue thereafter until the alert status is set to Inactive.
•
Inactive The Alert Definition is retained in the caDSR, but is not processed by the Automated Report Generation Process until it is made Active. If you set an alert as Inactive, you must provide a brief description explaining why the alert exists but is not active.
Midnight is used for all comparisons, e.g. 03/01/2006 to 03/02/2006 is the 24 hour period beginning midnight 03/01/2006.
