Using the caDSR Sentinel Tool > Monitors Tab
Monitors Tab
The Monitors Tab contains attributes controlling the Alert Report triggers.
All lists on this page support multiple selections using standard selection key combinations using the mouse, Ctrl and Shift.
Workflow Status(es)
Trigger an Alert Report when the Workflow Status is changed to one of the selections.
Registration Status(es)
Trigger an Alert Report when the Registration Status is changed to one of the selections.
Version
Trigger an Alert Report when the Version is changed to one of the selections.
Special selections:
Any Change
Triggers a report for any and all value changes to the record property.
Ignore
Disables triggers for Alert Report creation for the record property.
Whole Number Change
Applies to Version number changes only, triggers a report when the record whole number part changes, decimal number part changes are ignored.
Specific
Applies to Version number changes only, used to indicate a specific version number to trigger the report creation.
The selection combination on the Monitors Tab triggers the Alert Report creation by interrogating the data activities. When all attributes are set to "Any Change", all activity within the scope defined on the Criteria Tab will trigger the report creation. This is the default setting for new Alert Definitions. Selecting specific values for one or more attributes significantly limits that data activities triggering report creation because a specific activity matching the attributes values must be found in the caDSR history for the reporting dates. For example, selecting the Workflow Status "RELEASED" will only trigger a report when this specific change is found in the history - other changes would not trigger a report creation.
Some data is not logged in the caDSR history and will appear as "Details not available" on an Alert Report.
See the Report Output section for more details.
