Using the caDSR Sentinel Tool > Properties Tab
Properties Tab
The Properties Tab contains the basic Alert Definition properties.
Alert Name
The Alert Definition identification used both in the notification e-mail and the report body. Use names to briefly describe the function or purpose of the Alert, for example, "Changes to CCR" and "Watch for Retired Records".
Summary
Provides an overview of the Criteria and Monitors. Any reports generated, based on the current report definition, will only include information that meets the description shown.
Auto Run Options
Control the Alert Definition eligibilty in the Automated Report Generation Process by Frequency and Status. Running the Alert manually ignores both Frequency and Status.
Frequency
The execution cycle in the Automated Report Generation Process. The process runs daily at a time determined by the System Administrator. Alerts marked "Daily" are evaluated with each run. Those marked "Weekly" are evaluated only on the specific day of the week selected. Alerts marked "Monthly" are evaluated only on the specific day of the month selected. To indicate the first day of the month, select "1". To indicate the last day of the month, select "31". The Alert may be run manually at anytime via the Run button.
Status
Controls the Auto Run Frequency application in the Automated Report Generation Process.
Active, no end date
The Alert Definition is processed according to the Frequency.
Active until the first report …
After sending a report containing some activity, the status automatically changes to Inactive.
Active only from …
The Alert Definition is processed only within the dates specified. When the begin date is blank, the Alert Definition creation date is used. When the end date is blank and after reaching the begin date, the effect is the same as "Active, no end date".
Midnight is used for all comparisons, e.g. 03/01/2006 to 03/02/2006 is the 24 hour period beginning midnight 03/01/2006.
Inactive
The Alert Definition is retained in the caDSR and not processed by the Automated Report Generation. A brief description is required.
